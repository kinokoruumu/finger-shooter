# ゲームアーキテクチャ

## レイヤー構成

画面は3層構造。すべて `pointer-events: none` で手ジェスチャー専用。

1. **最背面**: `<video>` タグ（カメラ映像、CSS `scaleX(-1)` でミラー）
2. **中間**: Phaser Canvas（`transparent: true`、ターゲット・エフェクト描画）
3. **最前面**: React オーバーレイ（照準、スコア、UI）

## MediaPipe ↔ Phaser 分離

2つのループを統合しない。共有状態を介した非同期構造にする。

```
MediaPipe rAF → 共有状態に書き込み → Phaser update() で読み取り
```

### 共有状態の内容

- 照準座標 (x, y) — スムージング済み
- ガンポーズ中フラグ (boolean)
- 発射イベントキュー

## 座標変換

- ミラー変換 `x = 1 - normalized_x` はトラッキング層で **1箇所のみ** 行う
- 以降のゲーム層・UI層ではミラー済み座標を扱う
- 変換が複数箇所に散らばるとバグの温床になる

## ジェスチャー仕様

### ガンポーズ判定

以下をすべて満たす場合にガンポーズ：
- 人差し指が伸展（先端が第二関節より上）
- 中指・薬指・小指が屈曲（先端が第二関節より下）
- 親指が横方向に伸展

### 照準

- 人差し指 MCP(Landmark 5) → Tip(Landmark 8) のベクトルを延長した点
- 延長倍率は調整可能パラメータ
- 直近Nフレームの移動平均でスムージング

### 発射トリガー

- 親指先端(Landmark 4) と人差し指根元(Landmark 5) のユークリッド距離が閾値以下
- y座標のみの判定は禁止（手の傾きで誤検知する）
- 発射後 500ms 以上の再発射インターバル必須

## パラメータ管理

以下はハードコードせず設定オブジェクトで管理する：

| パラメータ | 概要 |
|---|---|
| 照準ベクトル延長倍率 | 人差し指の向きの延長度合い |
| スムージングフレーム数 | ブレ低減 vs 追従遅延のバランス |
| 発射距離閾値 | 正規化座標での距離 |
| 発射インターバル | 連射防止（最小500ms） |
| 命中判定半径 | 描画サイズより大きめ推奨 |
| 的の出現頻度・速度 | 難易度調整 |

## ターゲット仕様

### 地上ターゲット

- 地面の特定x座標から出現、放物線で飛び上がり→停滞→落下
- タイムアウト消滅 = ミス

### 電車ターゲット

- 画面右端からスポーン、一定速度で左移動
- 車体側面に複数の的（電車に追従、毎フレーム座標再計算必須）
- 画面外に出た時点で残存する的はすべてミス

## パフォーマンス

- ターゲット fps: 30（60fps だと端末が詰まる）
- カメラ解像度: `{ width: 640, height: 480 }` に制限
- MediaPipe 初期化は非同期（数秒）→ ローディングUIを表示
